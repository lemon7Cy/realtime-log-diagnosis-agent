# 面试讲法：实时日志异常诊断 Agent

## 一句话

这是一个可观测性诊断 Agent：日志流触发异常检测后，Agent 通过日志、指标、部署、拓扑和资源工具收集证据，输出带 ReAct 轨迹的根因报告。

## 值得讲的工程点

- 先用轻量检测器过滤日志流，只有异常才触发 LLM/启发式 Agent，控制成本和延迟。
- 工具层通过 DataSource 协议抽象，sample 数据、Prometheus、日志平台和部署平台可以替换。
- 告警有生命周期状态，状态变更和诊断轨迹通过 WebSocket 推送。
- 无模型 Key 时启发式 Agent 仍可复现完整链路，配置模型后自动升级为 LLM ReAct。

## 生产化权衡

- 诊断报告强调证据链和置信度，不把 LLM 输出当成最终事实。
- 接入真实平台时，工具调用需要超时、重试和权限隔离；报告生成失败不应丢失原始告警。
